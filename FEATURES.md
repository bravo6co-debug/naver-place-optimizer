### 1.1 OpenAI GPT-4 기반 키워드 생성

**설명:**
- GPT-4를 사용해 업종, 지역, 특성을 분석하여 35개 키워드 자동 생성
- Level 1-5 난이도별로 자동 분류
- 롱테일, 니치, 중간, 경쟁, 최상위 키워드를 균형있게 생성

**구현 위치:**
- `backend/services/keyword_generator.py`
- `backend/integrations/openai_api.py`

**필요 조건:**
- ✅ OpenAI API 키 (필수)
- 💰 비용: $0.05-0.10 / 분석 1회

**장점:**
- 🎯 사람이 생각하지 못한 키워드 발굴
- 🚀 35개 키워드를 10초 내 생성
- 🧠 업종별 맞춤 키워드 (음식점, 카페, 미용실 등 9개 업종)

**단점:**
- 💸 API 호출마다 비용 발생
- 🌐 인터넷 연결 필수
- ⚠️ 가끔 부정확한 키워드 생성 (10% 이하)

**의존성:**
- 이 기능 없으면 → 수동으로 키워드 입력 필요

**판단:**
- [ ] 불필요 - 키워드를 직접 입력하겠습니다
- [ ] 필요 - AI 자동 생성이 필요합니다

### 1.2 5단계 난이도 시스템 (Level 1-5)

**설명:**
- 키워드를 5단계 난이도로 분류
- Level 5 (롱테일) → Level 1 (최상위 경쟁)
- 각 레벨별 목표 순위, 달성 기간, 전략 제시

**난이도 체계:**
| Level | 이름 | 검색량 | 경쟁도 | 목표 순위 | 달성 기간 |
|-------|------|--------|--------|-----------|-----------|
| 5 | 롱테일 | 100↓ | 10↓ | Top 1-3 | 1-2주 |
| 4 | 니치 | 500↓ | 30↓ | Top 5 | 1개월 |
| 3 | 중간 | 1000↓ | 100↓ | Top 10 | 2-3개월 |
| 2 | 경쟁 | 5000↓ | 500↓ | Top 20 | 6개월 |
| 1 | 최상위 | 10000+ | 1000+ | 노출 목표 | 장기 |

**구현 위치:**
- `backend/models/keyword.py` - KeywordLevel Enum
- `backend/services/competition_analyzer.py` - 난이도 점수 계산
- `backend/services/strategy_planner.py` - 레벨별 전략

**필요 조건:**
- 없음 (독립 시스템)

**장점:**
- 🎯 현실적인 목표 설정 가능
- 📈 단계별 성장 전략 수립 용이
- 🏆 초보자도 쉽게 이해

**단점:**
- 없음 (핵심 로직)

**의존성:**
- 이 기능 없으면 → 모든 키워드를 동일하게 취급 (비효율)

**판단:**
- [ ] 불필요 - 레벨 구분 없이 사용하겠습니다
- [ ] 필요 - 난이도별 전략이 필요합니다

### 1.3 네이버 검색광고 API 통합 (실시간 검색량)

**설명:**
- 네이버 검색광고 API를 통해 **실제 검색량** 데이터 조회
- PC/모바일 검색량 분리 제공
- 3단계 경쟁도 (낮음/보통/높음) 제공
- Level 1-2 키워드만 API 호출 (비용 절감)

**구현 위치:**
- `backend/integrations/naver_search_ad_api.py`
- `backend/services/search_volume_estimator.py`

**필요 조건:**
- 네이버 검색광고 계정 (무료 가입)
- ✅ API 키 3종:
  - `NAVER_SEARCH_AD_CUSTOMER_ID`
  - `NAVER_SEARCH_AD_API_KEY`
  - `NAVER_SEARCH_AD_SECRET_KEY`
- 💰 비용: **무료** (API 호출 무료, 광고 집행 불필요)

**장점:**
- ✅ **실제 검색량** 데이터 (추정이 아님!)
- 📊 PC/모바일 분리 데이터
- 🎯 경쟁도 3단계 제공
- 💯 신뢰도 S등급 (가장 정확함)

**단점:**
- 🔑 API 키 발급 절차 복잡 (약 30분 소요)
- 📝 네이버 검색광고 계정 필요 (광고비 집행 불필요)
- ⏱️ API 응답 속도 느림 (키워드당 0.5-1초)

**폴백 시스템:**
- API 실패 시 → 인구 통계 기반 추정으로 자동 전환
- 서비스 중단 없음

**의존성:**
- 이 기능 없으면 → 인구 통계 추정치 사용 (신뢰도 B등급)

**판단:**
- [ ] 불필요 - 추정치로 충분합니다
- [ ] 필요 - 정확한 검색량 데이터가 필요합니다

### 1.4 인구 통계 기반 검색량 추정 (폴백)

**설명:**
- 행정안전부 인구 API + 업종별 이용률로 검색량 추정
- 네이버 검색광고 API 실패 시 자동 사용
- 12.5MB 음식점 통계 데이터 활용 (KC_618_LLR_RSTRT_CNBAS_TRND_2024.csv)

**추정 공식:**
```
검색량 = 지역 인구 × 업종 이용률 × 검색 행동률 × 레벨별 가중치
```

**구현 위치:**
- `backend/integrations/mois_population_api.py` - 인구 데이터
- `backend/integrations/restaurant_stats_loader.py` - 음식점 통계
- `backend/services/search_volume_estimator.py` - 추정 로직

**필요 조건:**
- 없음 (데이터 파일 포함됨)
- 💰 비용: 무료

**신뢰도 등급:**
- 📊 인구 데이터 있음 → **B등급** (신뢰도 보통)
- 📉 인구 데이터 없음 → **C-F등급** (인구 규모별 하향 조정)

**장점:**
- 💯 API 키 불필요
- 🚀 응답 속도 빠름 (즉시)
- 🔄 네이버 API 장애 시 자동 폴백

**단점:**
- ⚠️ 추정치이므로 실제와 차이 있음 (±30% 오차)
- 📊 지역 인구 데이터 없으면 정확도 하락

**의존성:**
- 네이버 검색광고 API의 폴백 시스템

**판단:**
- [ ] 불필요 - 폴백 없이 API만 사용

Phase 3: 중위권 진입 (3-6개월)
  - Level 3 키워드 5개
  - 예상 유입: +140명/일 (70%)

Phase 4: 상위권 도전 (6개월+)
  - Level 2 키워드 3개
  - 예상 유입: +200명/일 (100%)

- 🚀 초보자도 따라하기 쉬움
